# Documentation Rules

Last Updated: `<YYYY-MM-DD>`

## Documentation Quality Standard
Good documentation in this system is:
- Human-readable
- Concise
- Structured
- Current
- Reusable
- Reviewable by humans even when AI-generated

## Writing Rules
- Use clear headings and short sections.
- Prefer bullet points and checklists over long prose.
- Use explicit placeholders for project-specific content.
- State assumptions and open questions.

## Maintenance Rules
- Update docs when behavior, decisions, or workflow rules change.
- Mark significant changes with date and rationale.
- Remove stale or duplicated content.

## Reusability Rules
- Keep templates generic and technology-agnostic where possible.
- Separate policy documents from project-instance values.
- Preserve stable structure so future projects can copy with minimal edits.

## AI Usage Rules for Docs
- AI may draft and refactor docs.
- Humans validate correctness and intent.
- Final wording should optimize team understanding, not model parsing.

## Definition of Done for Doc Updates
- [ ] Relevant file updated
- [ ] Placeholders used where specifics are unknown
- [ ] Language remains concise and scan-friendly
- [ ] Linked artifacts remain consistent
