# AI Roles

Last Updated: `<YYYY-MM-DD>`

## Product AI
- Use When: discovery, problem framing, goal definition.
- Inputs: product context, ideas, user feedback.
- Outputs: refined problem statements, goal options, KPI suggestions.

## Planning AI
- Use When: backlog shaping, sequencing, sprint preparation.
- Inputs: epics/stories, constraints, team capacity.
- Outputs: story breakdowns, dependency maps, readiness gaps.

## Meeting AI
- Use When: daily/weekly syncs and review prep.
- Inputs: updates, blockers, completed work notes.
- Outputs: concise summaries, action lists, unresolved questions.

## Code Understanding AI
- Use When: onboarding, impact analysis, change planning.
- Inputs: codebase + architecture docs + story context.
- Outputs: component maps, impact/risk notes, implementation options.

## Implementation AI
- Use When: coding tasks with clear acceptance criteria.
- Inputs: story/task, architecture, constraints, tests.
- Outputs: scoped implementation drafts and change explanations.

## Review AI
- Use When: PR review and pre-merge checks.
- Inputs: diff, acceptance criteria, NFRs.
- Outputs: risk-focused review notes and test gap detection.

## Documentation AI
- Use When: updating docs after decisions/changes.
- Inputs: latest decisions and implementation outcomes.
- Outputs: concise human-readable doc updates.

## Workflow AI
- Use When: process health checks and operational consistency.
- Inputs: backlog state, reporting artifacts, cycle data.
- Outputs: bottleneck insights, process improvement suggestions.

## Role Routing Guide
- Discovery uncertainty -> Product AI
- Planning complexity -> Planning AI
- Communication load -> Meeting AI
- Code ambiguity -> Code Understanding AI
- Execution task -> Implementation AI
- Merge gate -> Review AI
- Knowledge drift -> Documentation AI
- Process drift -> Workflow AI
